# RAG: Визначення та підходи до retrieval

## Що таке RAG?

**Retrieval-Augmented Generation (RAG)** — це техніка, яка розширює можливості великих мовних моделей (LLM), дозволяючи їм отримувати та використовувати інформацію з зовнішніх джерел даних. RAG вирішує основні обмеження LLM:
- **Обмежена база знань**: Моделі можуть використовувати тільки свої попередні навчальні дані
- **Галюцинації**: Генерація неправильної інформації
- **Контекстне вікно**: LLM можуть обробити лише обмежену кількість тексту

RAG дозволяє організаціям доповнити LLM своїми власними даними без переучування (що коштує мільйони доларів і займає місяці).

## Архітектура RAG

RAG складається з двох основних компонентів:

### Retriever (Пошукова система)
- Сканує базу знань з використанням пошуку схожості (similarity search)
- Витягує найбільш релевантні документи/вектори для відповіді на запит
- Використовує **векторні бази даних** для збереження та пошуку вкладень тексту (embeddings)

### Generator (Генератор)
- LLM отримує релевантні документи від retriever
- Використовує цю інформацію як контекст для генерації точної відповіді

## Головні підходи до Retrieval

### 1. **Базові методи пошуку**
- **Keyword search** (ключовий пошук): Пошук точних слів/фраз у документах
- **Vector similarity**: Використання семантичної схожості через вектори embeddings
- **Hybrid search**: Комбінація ключового та векторного пошуку

### 2. **Оптимізація індексування**
- **Chunking**: Розділення документів на менші частини (речення, параграфи) для кращого пошуку
- **Small-to-big retrieval**: Пошук малих chunk'ів з подальшим витягуванням найбільшого контексту (65% покращення над базовим chunking)
- **Metadata filtering**: Фільтрація по метаданим для зменшення простору пошуку

### 3. **Просунуті техніки (Advanced RAG)**

**Pre-retrieval стратегії:**
- Трансформація запиту (query rewriting)
- Розширення запиту

**Intra-retrieval методи:**
- **Cross-encoder reranking**: Перепорядкування отриманих документів за релевантністю (59% покращення MRR@5)
- **Corrective RAG**: Коригування вибору документів на основі ймовірностей моделі

**Post-retrieval стратегії:**
- Рефайнинг результатів
- Різні методи реранкування

### 4. **Альтернативні підходи**
- **Hierarchical node-based systems**: Навігація по структурованому табеляцію змісту без vector embeddings (корисно для складних документів типу SEC filings)
- **Agentic RAG**: Використання агентів для більш розумного вибору документів

## Процес ROI

1. **Indexing**: Завантаження документів → розділення на chunks → преобразование в embeddings → зберігання у векторній базі
2. **Query processing**: Векторизація запиту користувача
3. **Retrieval**: Пошук схожих документів через similarity search
4. **Generation**: LLM генерує відповідь з урахуванням отриманого контексту

## Ключові переваги

✓ Покращена точність відповідей та фактична коректність
✓ Зменшення галюцинацій
✓ Доступ до актуальних даних без переучування моделі
✓ Гнучкість та масштабованість
